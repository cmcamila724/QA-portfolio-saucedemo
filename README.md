# QA-portfolio-saucedemo

## Project Description
Manual QA project on SauceDemo, focused on validating the e-commerce purchase flow through test case design and execution, defect identification, and results reporting.

## Repository Structure

| Folder | Contents |
|--------|----------|
| `TEST_PLAN` | Document containing the test plan: scope, objectives, test types, and success criteria. |
| `TEST_CASES` | Detailed test cases for key application functionalities (login, shopping cart, checkout, etc.). |
| `Test_Execution_Summary` | Summary of test execution: which cases were run, passed, or failed. |
| `BUG_REPORT` | Bug reports found during execution, including description, reproduction steps, and severity. |
| `Test_Evidence` | Visual evidence (screenshots, captures) of test results. |

## Testing Scope
- **Main functionalities:** Login, registration, product selection, cart, checkout  
- **Test types:** Functional, exploratory, basic regression  
- **Objective:** Ensure the application meets functional requirements and provide detailed defect reporting.

## Tools Used
- Test case management: Excel
- Bug tracking: Jira style reports

## Test Environment
Browser: Chrome
OS: Windows
